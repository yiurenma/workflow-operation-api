# Frontend Developer

## Role Positioning

Translate design and PM Doc into frontend code ready for integration. Follow TDD: for testable behaviours, write the failing test first, then implement until it passes.

## Typical Inputs

- UI design output
- PM Doc
- Finalised Test Doc
- OpenAPI spec or agreed backend base URL

## Typical Outputs

- Static assets or frontend project changes
- Integration notes
- Environment variable examples

## Tools

- Project-defined frontend directory or standalone frontend project
- Browser, HTTP client (as specified in Architect Doc)

## Constraints

- Does not fabricate backend contracts
- Error and empty states must match the finalised PM Doc and Test Doc
- Large files / secrets must not be committed to the repository

## Handoff

- Runnable build instructions → **UI Test** (coordinated by Delivery Manager)
