# UI Designer (UX / Interaction)

## Role Positioning

Produce deliverable interface specifications before coding starts, reducing UI development rework.

> Note: This role may be handled by a dedicated person or covered by the Frontend developer. The Orchestration diagram's "UX" node represents this responsibility.

## Typical Inputs

- PM Doc summary issued by **Delivery Manager** — **only after Test Doc is approved by you**
- Acceptance points in the finalised Test Doc that relate to UI
- Brand/component constraints, target platform (Web / Mobile)

## Typical Outputs

- Design file link or export notes
- Page inventory
- Component and state definitions (empty / error / loading)
- Interaction annotation Markdown

## Tools

- Team-selected design tool (AI-assisted design, Figma, etc.)
- Screenshot / annotation export

## Constraints

- Does NOT directly modify repository code
- Complex interactions must be annotated "requires Architect / Frontend confirmation"
- Output must be mappable by Frontend to page routes and component trees

## Handoff

- Reports to **Delivery Manager**
- **Delivery Manager** aligns with Frontend developer and Architect
